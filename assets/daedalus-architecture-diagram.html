Daedalus Architecture

Durable orchestration for agentic SDLC lanes — shadow → active → ship

Daedalus Engine

  • • Runtime Loop: Tick → Ingest → Derive → Dispatch → Record
  • • Leases: Heartbeat-based ownership with automatic recovery
  • • Shadow Mode: Safe observation without side effects
  • • Active Mode: Real execution with failure tracking & retries
  • • Operator Surfaces: status, doctor, watch, shadow-report, active-gate

Workflow Wrapper

  • • Policy Brain: Decides nextAction, health, review state
  • • Actors: Internal Coder (Codex), Internal Reviewer (Claude), External Reviewer (Codex Cloud)
  • • State Machine: implementing → review → publish → merge
  • • Handoffs: Explicit, durable, survive restarts
  • • Semantic → Execution: Translates policy into dispatchable actions

State & Storage

  • • SQLite: Canonical runtime state — lanes, actions, reviews, failures, leases
  • • JSONL: Append-only event history for audit & replay
  • • Lane State: .lane-state.json + .lane-memo.md per workspace
  • • Hot-reload: workflow.yaml edits take effect on next tick
  • • Idempotency: Composite keys prevent duplicate actions

External Integrations

  • • GitHub: Issues, PRs, review threads, comment publishing
  • • Webhooks: Slack, HTTP JSON with SSRF guard
  • • Runtimes: claude-cli, acpx-codex, hermes-agent
  • • Models: gpt-5, claude-sonnet-4-6, gpt-5.4
  • • Config: workflow.yaml drives all behavior

Supervision & Ops

  • • systemd: daedalus-active@.service for 24/7 supervision
  • • Watch TUI: /daedalus watch — live lanes + alerts + events
  • • HTTP Status: localhost:8765 for dashboards & health checks
  • • Stall Detection: Automatic termination of wedged workers
  • • Alerts: Outage detection with deduplication

Design Principles

  • • Wrapper decides what. Daedalus decides how to orchestrate it durably.
  • • State is tracked, not guessed. SQLite is current truth.
  • • Bad edits don't crash. workflow.yaml errors are ignored until fixed.
  • • Recovery is automatic. Dead workers never block forward motion.
  • • Humans observe, don't schedule. Passive by default, intervene only when needed.
